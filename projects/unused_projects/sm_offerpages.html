    Offer Pages


    "The simplicity of the site. Just enough information presented and very easy to navigate."


    Design Statement

    Re-define the user experience of the offers/savings page for the product.


    Problems with current page

    1. The navigation to other offers in the same product type was un-intuitive.
    2. The value proposition and/or savings for the offer were less apparent and hidden under "Your Savings".
    3. Tracking: Since all the offers were in one page, current implementation of the page made it difficult to track what offers were being interacted with.

    Motivations

    1. Transparency: Expose what the user will pay and how much (if at all) would the user save with the recommended offer.
    2. Comparison: Better visual way to compare all available offers.
    3. Edge Cases: Data analysis revealed that around 40–50% of the user base will eventually see any offer. Others might either not be eligible for any offer or not have savings. The next gen designs should take care of as much of the edge cases as possible while leaving enough leeway for users to explore on their own.
    4. Modular: De-constructing the architecture of the page which lets efficient implementation of all use cases while allowing the partners modify the marketing text and disclosure around the offers.

    Timeline

    The project kickstarted as a partner request to support "Student Loans" category. This was undertaken as an opportunity to redo the "Savings" area with a more consolidated page per offer type.


    Outcome

    This project is still ongoing with iterative changes and enhancement to features. As of Q3 2018, the improved functionality and UI was in production for Auto Loans, Student Loans, Personal Loans, Credit Cards, Home Loans and a generic offer page for all other offer types, both web and mobile.

     

     


    Process

    These pages were built one by one, such that certain nuances particular to the product type could be addressed properly. The process for building these page involved:


    1. Understanding the space of each product due to unfamiliarity to the finance sector.
    2. Looking at the user data to understand the use cases and edge case scenarios.
    3. Building task-flows , UI architecture and low-high fidelity mockups
    4. Testing with Partners for initial feedback to understand if the concept matched the industry standards.
    5. Building semi- functional prototype and testing for usability, user-satisfaction, aesthetic appeal and ease of use.
    6. Designing the UI and handing off the project to the developers with detailed requirements, model calculators and UI specification.

    Since the pages were built one-by-one, this was truly an iterative project. Every next product page was informed by the performance of the previous pages and the improvements were implemented across all the pages.


    Take Away

    Being an outsider to the fin-tech industry, this project helped me get the exposure to the world of financial institutions, the nuances of how, for example, credit cards or Auto refinance works, what factors are important for the lenders and what goes into the process of loan underwriting.


    From the technical perspective, I learned the importance of consistency across different components and pages of the product, both in terms of design and code. 


    It is easy to design and build in a silo when you are focussed on task based output, losing the sight the bigger picture. It is then when you overstep consistency for getting the work done.

    This is when I took a step back, evaluated all pages for inconsistencies. This led way to setup Design System at SavvyMoney.
